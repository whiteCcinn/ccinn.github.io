k8s の 基础知识

2019-05-16

k8s 知识点一

k8s 总架构由于 2 个概念构成

  • Master
  • Node

Master 和 Node 都是服务器物理机或者虚拟机, 都可以由于一台或者多台组成，如果 Master 不考虑HA(高可用)的情况下。一般都是 1 台 Master + N 台 Node。

Master 上所需要的服务

  • etcd
  • Api Server
  • Controller Manager
  • Scheduler

后三个组件构成了 Kubernetes 的总控中心，这些进程实现了整个集群的资源管理、Pod 调度、弹性伸缩、安全控制、系统监控和纠错等管理功能，并且全都是自动完成。

Node 上所需要的服务

  • kubelet
  • proxy
